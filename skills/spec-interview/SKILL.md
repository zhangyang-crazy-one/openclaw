# Spec Interview Skill - 需求访谈与规格对齐

*Version: 1.0.0*
*Created: 2026-02-21*

---

## 概述

**Spec Interview** 是一个专业的需求对接技能，用于：
- 需求访谈 (Requirements Elicitation)
- 规格定义 (Specification Definition)  
- 颗粒度对齐 (Granularity Alignment)
- 期望管理 (Expectation Management)

## 核心原则

### 1. 访谈技巧

| 原则 | 描述 |
|------|------|
| **开放式问题** | 避免"是/否"问题，使用"什么"、"为什么"、"如何" |
| **积极倾听** | 重述用户观点确认理解 |
| **澄清追问** | 深入挖掘细节和背景 |
| **场景还原** | 通过具体场景理解抽象需求 |

### 2. 颗粒度评估

**需求颗粒度层级:**

```
Level 1: 战略层 (Strategic)
  - 业务目标
  - 核心价值
  - 成功标准

Level 2: 战术层 (Tactical)
  - 功能模块
  - 用户旅程
  - 数据流程

Level 3: 操作层 (Operational)
  - 具体功能点
  - 交互细节
  - 边界条件
```

### 3. 需求验证问题

**必问的核心问题:**

1. **目标问题**
   - "你想解决什么核心问题？"
   - "成功的标准是什么？"
   - "谁会从中受益？"

2. **范围问题**
   - "这个需求包含什么？不包含什么？"
   - "有什么约束条件？"
   - "优先级是什么？"

3. **细节问题**
   - "能给我一个具体的使用场景吗？"
   - "现在的痛点是什么？"
   - "你期望的输出是什么？"

4. **验证问题**
   - "如果做完了，你怎么知道成功了？"
   - "有什么量化指标？"

## 访谈流程

### Step 1: 开场 (5分钟)

```
1. 介绍自己和技能目的
2. 说明访谈方式
3. 确认时间安排
```

**开场话术:**
> "你好！我是 Curious，一个专注于深度研究的AI助手。今天我想和你进行一次需求访谈，帮助我更好地理解你的期望。我的目标是：**把模糊的想法转化为可执行的规格**。

### Step 2: 背景探索 (10分钟)

```
1. 了解业务背景
2. 识别关键利益相关者
3. 理解当前痛点
```

**背景问题:**
- "能告诉我这个需求的背景吗？"
- "之前是怎么解决这个问题的？"
- "为什么现在需要做这个？"

### Step 3: 需求深挖 (15分钟)

```
1. 核心需求拆解
2. 用户场景分析
3. 约束条件识别
```

**深度问题:**
- "能描述一下你期望的完整流程吗？"
- "在这个场景下，你会怎么做？"
- "如果X情况发生，你会怎么处理？"

### Step 4: 规格确认 (10分钟)

```
1. 需求优先级排序
2. 验收标准明确
3. 颗粒度对齐
```

**确认问题:**
- "如果只能实现3个功能，你会选哪3个？"
- "怎么算'做完了'？有什么标准？"
- "你希望的交付形式是什么？"

### Step 5: 总结 (5分钟)

```
1. 重述关键点
2. 确认理解
3. 约定下一步
```

**总结话术:**
> "让我确认一下我理解的是否正确：1)... 2)... 3)..."

## 需求模板

### 功能需求模板

```markdown
## [功能名称]

### 背景
- 用户故事: "作为...我希望...以便..."

### 需求描述
1. 核心功能点
2. 扩展功能点
3. 边界条件

### 验收标准
- [ ] 标准1: ...
- [ ] 标准2: ...

### 优先级
- P0 (必须)
- P1 (重要)
- P2 (可选)

### 约束
- 技术约束
- 时间约束
- 资源约束
```

### 非功能需求模板

```markdown
## [质量属性]

### 性能要求
- 响应时间: < X ms
- 并发数: > Y

### 可用性要求
- 可用性: 99.9%
- 故障恢复: < X 分钟

### 安全要求
- 认证方式: ...
- 数据加密: ...
```

## 常见问题应对

| 场景 | 应对策略 |
|------|----------|
| 需求太模糊 | "能给我一个具体例子吗？" |
| 范围蔓延 | "这个需求和之前提到的X是什么关系？" |
| 互相矛盾 | "我注意到A和B可能冲突，你怎么看？" |
| 过于细节 | "先聚焦在'做什么'，'怎么做'我们后面再讨论" |
| 过于抽象 | "在实际使用中，这个功能会怎么被用到？" |
| 优先级不清 | "如果只能做一个，你会选哪个？" |

## 使用方法

### 在对话中使用

当用户提出一个需求时：

1. **识别需求类型** - 功能/非功能/技术
2. **评估颗粒度** - 太粗/适中/太细
3. **应用访谈技巧** - 提问→倾听→确认
4. **输出结构化** - 使用上述模板

### 示例对话

**用户**: "我想做一个股票分析系统"

**AI (应用Spec Interview)**:

> "听起来很有意思！让我帮你把想法细化。

**Step 1 - 背景了解:**
- "能告诉我为什么想做这个系统吗？"
- "是为自己投资还是机构？"

**Step 2 - 需求深挖:**
- "你希望的'分析'具体包含什么？基本面？技术面？"
- "主要分析哪些股票？A股？美股？"
- "现在有什么痛点吗？"

**Step 3 - 规格确认:"
- "如果只能实现3个功能，你会选哪3个？"
- "怎么算'分析完成'？"

---

这样我可以把你的想法转化为完整的规格说明。"
